 MODULE 1: ORGANIZATION / MULTI-TENANCY (MISSING — CRITICAL)
================================================================================

Status: NOT IMPLEMENTED — This is the #1 priority module.

Currently there is only a "Workspace" model (workspaces table) which has:
  name, slug, description, owner_id, logo_url, settings

What is MISSING:

  [MISSING] 1.1 — Organization Table
    Need a proper "organizations" table with:
      - organization_id (PK)
      - name
      - logo_url
      - tax_id
      - address (street, city, state, country, zip)
      - phone
      - email (contact email)
      - website
      - industry/sector
      - subscription_plan (for future package/pricing)
      - max_users (limit for plan)
      - max_projects (limit for plan)
      - is_verified (boolean)
      - is_active (boolean)
      - created_at, updated_at

  [MISSING] 1.2 — Organization Foreign Keys on ALL Core Tables
    The following tables need an "organization_id" column as FK:
      - users
      - departments
      - projects
      - teams
      - tasks
      - clients
      - workspaces
      - expenses
      - timesheets
      - support_requests
    Without this, ALL data is shared globally — anyone who registers
    can see everyone else's projects, tasks, users, etc.

  [MISSING] 1.3 — Data Isolation / Tenant Filtering
    Every API query must be scoped by the current user's organization_id.
    Currently there is ZERO data isolation. This is a launch-blocker.

  [MISSING] 1.4 — Organization Management Pages (Frontend)
    - Create Organization page
    - Edit Organization settings page
    - Organization profile/branding page
    - Organization admin dashboard

 MODULE 2: USER VERIFICATION & APPROVAL FLOW (MISSING — CRITICAL)
================================================================================

Status: NOT IMPLEMENTED

Currently the registration flow (backend/app/routers/auth.py) immediately
creates the user and returns a token. There is no verification of any kind.

  [MISSING] 2.1 — Email Verification
    - Add "email_verified" (boolean, default=False) column to users table
    - Send verification email with unique token on registration
    - Verification endpoint to mark email as verified
    - Block login if email_verified is False

  [MISSING] 2.2 — User Status / Admin Approval
    - Add "user_status" column to users table:
        Values: "pending" | "approved" | "rejected" | "suspended"
    - New users register with status = "pending"
    - Pending users CANNOT login (blocked at auth layer)
    - Only organization admin can approve/reject users
    - Email notification sent on approval/rejection

  [MISSING] 2.3 — Admin Approval Dashboard Page (Frontend)
    - Page for organization admins to see all pending registration requests
    - Approve / Reject buttons with confirmation
    - Filter by status (pending, approved, rejected, suspended)
    - Bulk approve/reject functionality
    - Email sent to user on status change

  [MISSING] 2.4 — Organization Selection During Registration
    - Registration form needs an "Organization" field
    - User selects which organization they're joining
    - OR enters an invite code / organization slug
    - Registration request goes to that org's admin for approval

 MODULE 4: TEAM CREATION ERROR (BUG)
================================================================================

Status: BUG — Error occurs when creating teams, even with empty teams

  [BUG] 4.1 — Team Creation Fails
    Location: Frontend — app/(protected)/teams/page.tsx
              Backend — backend/app/routers/teams.py

    Root cause analysis:
    - Backend create_team() requires "manager" or "admin" role (line 194)
    - The frontend sends empty string "" for department_id, lead_id,
      parent_team_id when none is selected — the backend may reject
      empty strings as invalid foreign keys
    - Need to convert empty strings to null/None before sending
    - The error from the screenshot shows "Not Found" on workspace
      creation — check if teams page is calling workspace API by mistake
      or if there's a routing conflict

    Fix needed:
    - Frontend: Convert empty strings to null for optional FK fields
    - Backend: Handle empty string FK values gracefully or return
      a clearer error message
    - Verify the user has the correct role (manager/admin)

 MODULE 5: SIDEBAR THEME BUG (BUG — UI)
================================================================================

Status: BUG — Sidebar shows white on first load

  [BUG] 5.1 — Sidebar Defaults to White Background
    Location: frontend/components/layout/Sidebar.tsx
              frontend/components/providers/ThemeProvider.tsx

    Root cause:
    - ThemeProvider uses defaultTheme="system" with enableSystem
    - Sidebar uses: isDark ? "bg-[#191919]/90" : "bg-gray-200/70"
    - On first load (before hydration), "mounted" is false so sidebar
      returns null, BUT the parent container likely has no background set
    - When mounted becomes true, theme reads from "system" which may
      report "light" even if user expects dark

    Fix needed:
    - Change ThemeProvider to defaultTheme="dark" (since app is dark-themed)
    - OR add suppressHydrationWarning and a CSS-based dark default
    - The sidebar should use CSS variables or className-based theming
      instead of conditional JS class to avoid the flash

 MODULE 7: SUBTASK DISPLAY ISSUES (BUG)
================================================================================

Status: BUG — Multiple subtask display issues

  [BUG] 7.1 — Subtasks Don't Show Parent Task in Task List
    Location: frontend/components/TaskListPage.tsx

    The All Tasks list view shows subtasks as regular top-level tasks.
    There is no indication of which parent task a subtask belongs to.

    Fix needed:
    - In the task list, show "Parent: [task name]" for subtasks
    - OR indent subtasks under their parent
    - OR add a "Parent Task" column to the task table
    - The Task model HAS parent_task_id but it's not displayed

  [BUG] 7.2 — Subtasks Not Showing After Creation Inside Task Detail
    Location: frontend/components/TaskDetailPanel.tsx (lines 464-563)

    Analysis of SubtasksSection:
    - Fetches subtasks via: /api/tasks?parent_task_id=${taskId}
    - Expects response format: { items: [...] }
    - After creating a subtask, it appends the raw API response to state
    - The issue may be that the backend /api/tasks endpoint does not
      filter by parent_task_id, or returns a different response format

    Fix needed:
    - Verify backend /api/tasks supports parent_task_id query parameter
    - Verify response format matches what frontend expects
    - After creating subtask, refetch the subtasks list instead of
      just appending (to ensure consistency)
    - The toggle uses "done" status but the app uses "completed" — mismatch

 MODULE 8: DROPDOWN / SELECT TEXT HIDDEN (UI BUG)
================================================================================

Status: UI BUG — Dropdown sometimes hides text (highlight issue)

  [UI] 8.1 — Dropdown Options Text Not Visible
    Location: Various pages (Projects, Tasks, Teams)

    From the screenshot showing project creation, dropdown options
    like "Draft", "On Hold", "Completed", "Archived" appear greyed
    out / barely visible. The "Active" option with highlight has
    a light blue background that makes text hard to read.

    Root cause:
    - Native <select>/<option> elements with dark theme styling
    - Option text color doesn't contrast well with selection highlight
    - Disabled options (Draft, Completed, Archived) are too faded

    Fix needed:
    - Use custom dropdown component instead of native <select>
    - OR explicitly set option background and text colors
    - Ensure contrast ratio meets WCAG standards
    - Add proper focus/hover states with visible text
